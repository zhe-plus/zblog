## 1. 项目概述

### 1.1 项目背景

在信息爆炸的时代，个人在日常学习、工作和生活中会持续产生和积累大量的信息碎片——阅读笔记、技术心得、项目文档、生活记录、灵感片段等。这些内容分散在各个平台（本地 Markdown 文件、云笔记、社交媒体、即时通讯记录等），导致以下痛点：

- **信息分散**：知识散落各处，缺乏统一检索入口，查找效率低下
- **格式不一**：不同工具存储格式各异，难以建立结构化的知识体系
- **无法沉淀**：零散的记录难以形成系统化的知识网络，回顾和复用困难
- **归属感弱**：依赖第三方平台，数据归属和隐私控制能力不足

基于以上痛点，zblog 定位为一个集 **个人知识库**、**个人记事本** 与 **个人资料管理** 于一体的综合型个人博客系统：

#### 1.1.1 个人知识库

面向长期知识沉淀场景，将分散的学习笔记、技术文章、读书心得等系统化整理：

- 支持 Markdown 富文本编辑，兼顾可读性与可移植性
- 通过分类体系与标签网络构建知识图谱，实现知识间的双向链接与关联推荐
- 提供全文搜索能力，快速定位历史积累的任意内容
- 支持文章版本历史，追踪知识的演进过程

#### 1.1.2 个人记事本

面向日常记录与快速写作场景，降低记录门槛：

- 支持快速草稿录入，随手记录灵感与想法
- 提供简洁无干扰的写作体验（类似纯文本编辑器）
- 支持按时间线回溯，像翻阅日记本一样回顾过往记录
- 可对记事内容设置私密/公开状态，灵活控制可见范围

#### 1.1.3 个人资料管理

面向各类参考资料的归档与检索场景：

- 支持文件上传与附件管理（图片、PDF、文档等）
- 提供链接收藏功能，将外部优质资源纳入统一管理
- 通过标签和分类对资料进行多维索引
- 支持批量导入导出，保障数据可迁移性

综上，zblog 并非传统意义上仅用于对外展示的博客，而是一个 **以博客形态承载个人知识全生命周期管理** 的工具平台——既是外部分享的窗口，也是内部知识沉淀的仓库。

### 1.2 项目目标

基于 1.1 所述背景与定位，zblog 项目确立以下核心目标：

| 编号 | 目标 | 说明 |
|------|------|------|
| G1 | **知识统一归集** | 将散落在各处的笔记、文章、资料汇聚至单一平台，提供统一检索入口 |
| G2 | **结构化知识体系** | 通过分类、标签、双向链接构建知识网络，让孤立内容产生关联价值 |
| G3 | **低摩擦写作体验** | 提供 Markdown 编辑器与快速草稿能力，降低记录与发布的心理门槛 |
| G4 | **资料一站式管理** | 整合文件存储、链接收藏、批量导入导出，形成完整的个人资料中心 |
| G5 | **公开与私密灵活控制** | 支持内容发布为公开文章或保留为私密笔记，兼顾分享与隐私 |
| G6 | **数据自主可控** | 数据存储在自有服务器，支持一键导出，不依赖第三方平台 |
| G7 | **轻量易部署** | 保持系统架构简洁，降低安装与运维成本，适合个人长期维护 |
| G8 | **SEO 友好** | 公开内容具备良好的搜索引擎可发现性，方便他人检索与引用 |

### 1.3 目标用户

zblog 的核心用户群体为 **技术开发人员**，具体画像如下：

**主要用户：**

- **后端/前端/全栈开发者**：日常积累技术笔记、踩坑记录、架构设计文档，需要结构化知识库辅助快速回顾与复用
- **DevOps / SRE 工程师**：记录运维脚本、故障排查流程、基础设施配置，需要可全文搜索的私有资料库
- **技术管理者 / 架构师**：撰写技术方案、设计文档、团队规范，需要版本历史追踪内容演进

**次要用户：**

- **技术内容创作者**：通过公开文章分享技术见解，建立个人品牌
- **计算机相关专业学生**：整理学习笔记、项目复盘，构建个人知识体系

**用户特征：**

- 熟悉 Markdown 语法，偏好键盘驱动的编辑体验
- 具备自主部署能力，倾向于数据自托管
- 对知识管理有结构化需求，习惯通过分类和标签组织信息
- 需要代码高亮、技术图表等开发者友好特性

### 1.4 术语与缩写

本文档中使用的关键术语定义如下：

| 术语 / 缩写 | 全称 | 说明 |
|-------------|------|------|
| **zblog** | — | 本项目名称，个人博客/知识管理系统 |
| **文章（Post）** | — | 系统核心内容单元，包含标题、正文、元数据，可设为公开或私密 |
| **页面（Page）** | — | 独立静态页面（如"关于"），与文章区别在于不参与时间线排序 |
| **草稿（Draft）** | — | 尚未发布的内容，仅博主可见，不对外展示 |
| **分类（Category）** | — | 文章的一级归类维度，一篇文章属于一个分类，用于构建知识树 |
| **标签（Tag）** | — | 文章的灵活标注维度，一篇文章可有多个标签，用于构建标签网络 |
| **双向链接（Backlink）** | — | 文章 A 引用文章 B 时，B 自动显示"被 A 引用"的关联，形成知识网络 |
| **前台（Frontend）** | — | 面向读者/访客的公开页面，即博客展示层 |
| **后台（Admin）** | — | 面向博主的管理界面，用于内容编辑与站点配置 |
| **Markdown** | — | 轻量级标记语言，系统核心编辑格式 |
| **SEO** | Search Engine Optimization | 搜索引擎优化，提升公开内容在搜索引擎中的排名 |
| **RSS** | Really Simple Syndication | 内容聚合标准，用于向订阅者推送更新 |
| **MVP** | Minimum Viable Product | 最小可行产品，即 v1.0 版本的交付范围 |
| **WYSIWYG** | What You See Is What You Get | 所见即所得编辑器，作为 Markdown 编辑的辅助模式 |
| **SSG** | Static Site Generator | 静态站点生成，可选的技术方案之一 |
| **CDN** | Content Delivery Network | 内容分发网络，用于加速静态资源加载 |
